# UI System — Technical Documentation

## Architecture

### shadcn/ui v4 + Base UI
This project uses `shadcn@4.x` which has migrated from Radix UI primitives to **Base UI** (`@base-ui/react`). Key difference:
- Old Radix pattern: `<Button asChild><Link /></Button>` (Slot composition)
- New Base UI pattern: `<Button render={<Link />}>` (render prop)

All custom components must use the `render` prop for polymorphic rendering. The `asChild` pattern will TypeScript error.

### Tailwind CSS v4
Tailwind v4 uses `@import "tailwindcss"` (not a `tailwind.config.js` file). The `@theme inline {}` block in `globals.css` maps CSS custom properties to Tailwind color utilities. No `tailwind.config.ts` is needed.

Color tokens are defined as `oklch()` values for:
- Perceptual uniformity (colors look equally vibrant across all hues)
- HDR display support
- Precise control over lightness and chroma

### Providers Architecture
The root layout (`src/app/layout.tsx`) is a React Server Component. It cannot use React context directly. The `Providers` component (`src/components/providers.tsx`) is a Client Component that wraps:
1. `SessionProvider` from next-auth — makes session available via `useSession()`
2. `QueryClientProvider` from TanStack Query — makes the query cache available
3. `Toaster` (Sonner) — renders toast notifications

The `QueryClient` is created inside a `useState` hook to ensure each user request gets a fresh client (prevents cache sharing between SSR requests).

### CSS Variables Theme
All design tokens are CSS custom properties defined in `:root` in `globals.css`. This means:
- Components read `var(--primary)` etc. without hardcoded colors
- Dark mode works by redefining the same variables under `.dark` class
- The `@theme inline` block makes these available as Tailwind utilities (`bg-primary`, `text-foreground`, etc.)

### Print Styles
`globals.css` includes `@media print` rules:
- `.no-print` — hides navigation, buttons, sidebars when printing
- `.print-only` — shows prescription-specific print headers
Used by the prescription print feature in Phase 7.

## `cn()` Utility
Located in `src/lib/utils.ts` (auto-generated by shadcn init):
```typescript
import { clsx, type ClassValue } from 'clsx';
import { twMerge } from 'tailwind-merge';

export function cn(...inputs: ClassValue[]) {
  return twMerge(clsx(inputs));
}
```
Use this everywhere to conditionally merge Tailwind classes. `twMerge` resolves conflicts (e.g., `bg-red-500 bg-blue-500` → `bg-blue-500`).
